# 小红书舆情获取 Skill

## 前置检查

首次使用时，需验证环境是否就绪。

### 第一步：工具可用性检查

```bash
opencli --version
opencli xiaohongshu --help
```

如未安装，引导用户：
```bash
npm install -g @jackwener/opencli
```

### 第二步：预检测试

执行一个最小化测试，确认可以访问小红书：

```bash
# 搜索一个公开的测试关键词（如产品名）
opencli xiaohongshu search "测试" --limit 3 --format json

# 尝试获取一个帖子的评论
opencli xiaohongshu comments "https://www.xiaohongshu.com/search_result/test" --limit 2 --format json
```

### 第三步：预检结果处理

| 预检结果 | 含义 | 处理方式 |
|----------|------|----------|
| ✅ 搜索成功、评论返回数据 | 环境就绪 | 进入需求确认 |
| ⚠️ 搜索成功但评论为空 | 可能未登录或受限 | 提示用户登录 Chrome |
| ❌ 完全失败（SECURITY_BLOCK） | 小红书风控拦截 | 切换 CDP 方案或引导用户 |

|------|---------|----------|
| **A. 广泛调研用户反馈** | 大量帖子 + 全量评论分析 | 产品刚上线、改版后、需要摸清整体口碑 |

## 数据采集

### 内容优先级

| 优先级 | 内容类型 | 说明 |
|--------|----------|------|
| 1 | **深度测评帖** | 真实体验长文，有具体使用场景和问题，非软广 |
| 2 | **高赞评论（≥50赞）** | 反映群众趋向，情感分析的锚点 |
| 3 | **官方/宣传帖** | 单独标注，不纳入用户舆情统计 |

### 帖子类型区分

| 类型 | 识别特征 | 处理方式 |
|------|----------|----------|
| **官方帖** | 品牌官方账号（如品牌名、品牌官方等） | 单独统计、标注来源 |
| **媒体/评测帖** | 第三方媒体、种草避雷指南 | 归类为"专业内容" |
| **普通用户帖** | 真实体验分享 | 情感分析主体 |

### 采集命令

```bash
# 搜索帖子（搜索结果已包含完整URL，可直接用于后续调用）
opencli xiaohongshu search "<关键词>" --limit <数量> --format json

# 获取帖子详情 - 必须使用完整URL（含xsec_token），禁止使用 bare note ID
opencli xiaohongshu note "<完整URL（含xsec_token参数）>" --format json

# 获取评论（关键：评论才是舆情金矿）
opencli xiaohongshu comments "<完整URL（含xsec_token参数）>" --limit <数量> --format json
```

### 搜索策略

| 调研目的 | 建议数量 | 分批策略 |
|----------|----------|----------|
| 快速验证（预检） | 3-5 个 | 单次搜索即可 |
| 初步摸底 | 10-20 个 | 单次搜索 |
| 深度调研 | 30-50 个 | 分 2-3 批，间隔 3-5 秒 |

**分批去重**：搜索结果可能包含重复帖子（尤其热门关键词），按 `url` 字段去重后再采集详情。

### 搜索与采集流程

```
1. 搜索关键词 → 获取帖子列表（含完整URL）
2. 去重 + 按点赞排序
3. 选择高价值帖子（高赞 > 深度内容 > 普通分享）
4. 依次获取详情 + 评论（每步间隔 2-3 秒）
5. 汇总分析
```

### 请求频率控制

| 场景 | 建议延迟 | 原因 |
|------|----------|------|
| 搜索 | 1-2 秒 | 搜索结果包含完整 URL，后续可直接使用 |
| 帖子详情 | 2-3 秒 | 避免风控拦截 |
| 评论 | 2-3 秒 | 评论数据敏感，需控制频率 |
| 并发 | 禁止并发 | 同一 IP 并发请求易触发 SECURITY_BLOCK |

### 数据校验

| 信号 | 含义 | 处理方式 |
|------|------|----------|
| `SECURITY_BLOCK` 错误 | 小红书风控拦截，URL 未携带有效 token | 换用搜索结果中的完整 URL 重试 |
| `NOTE_NOT_FOUND` | 帖子不存在或已被删除 | 跳过该帖子 |
| `EMPTY_RESULT` | 页面加载无内容 | 可能需要登录或帖子已受限 |
| 连续 3 篇获取失败 | 策略问题 | 降低频率、换用 URL 或切换方法 |

### 评论分析方法

评论是舆情分析的核心，但小红书评论有其特殊性，分析时必须结合帖子上下文。

**第一步：读帖子再读评论**
- 先获取帖子标题、正文、标签，了解帖子背景
- 再看评论，结合上下文判断真实情绪
- 同一句话在赞美帖和批评帖中的含义可能完全相反

**第二步：识别阴阳怪气**

| 评论特征 | 识别方式 | 处理 |
|----------|----------|------|
| 夸张语气 | 过度感叹、反讽句式 | 可能是讽刺 |
| 负数/亏损 | "赚了-XXX"、"亏了XXX" | 大概率自嘲，不是真负面 |
| 模仿炫富 | 模仿原帖夸张收入数字 | 讽刺原帖 |
| 正话反说 | 表面夸实际贬 | 需结合上下文 |

**第三步：去噪**

| 类型 | 特征 | 处理 |
|------|------|------|
| 软广/营销号 | 频繁推荐产品、带购买链接 | 单独标注，不计入舆情 |
| 纯炫富帖 | 晒收入、晒成果 | 需判断是否真实 |
| 杠精/引战 | 无具体观点只有情绪 | 忽略 |

**第四步：情绪归类**

| 真实情绪 | 典型表达 |
|-----------|----------|
| 正面 | 真诚感谢、具体认可、提出建设性意见 |
| 负面 | 具体批评、理性质疑、有理有据反对 |
| 中性 | 问技术问题、讨论成本、客观陈述 |

### 常见错误

| 错误 | 正确做法 |
|------|----------|
| 使用 bare note ID（如 `69b01dcf0000000022021dc4`） | 必须从搜索结果中获取完整 URL（含 xsec_token） |
| 未确认需求就开始爬 | 先问用户监测对象和目的 |
| 忽略官方帖和普通帖的区分 | 分开统计，以普通用户帖为主 |
| 只爬帖子不爬评论 | 评论才是舆情金矿 |
| 把宣传帖当真实反馈 | 识别软广、官方号，单独标注 |
| 预检失败后继续爬取 | 先解决环境问题（登录/工具）再继续 |
| 并发请求 | 禁止并发，同一 IP 并发易触发风控 |

## 舆情分析框架

### 分析维度

| 维度 | 分析内容 |
|------|----------|
| **情绪分析** | 正面/中性/负面占比，核心情绪标签 |
| **话题提取** | 高频话题、争议焦点、用户痛点 |
| **意见领袖** | 高赞评论用户（≥50赞）及其观点原文 |
| **趋势判断** | 时间线上的情绪变化 |

## 工具说明

**opencli 是首选工具**，因为：
- 内置小红书适配器（搜索、帖子、评论）
- 自动处理反爬机制
- 输出结构化 JSON

**CDP 仅作为补充**：
- opencli 完全失败时使用 CDP
- 处理登录墙或风控限制
