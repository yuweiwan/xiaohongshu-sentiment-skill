# 小红书舆情获取

## 核心原则

**评论才是舆情金矿。** 只爬帖子不爬评论等于只看到表面。分析评论时必须先读懂帖子背景——同一句话在赞美帖和批评帖中含义完全相反。

## 前置检查

首次使用验证环境：

```bash
opencli --version
opencli xiaohongshu --help
```

未安装：`npm install -g @jackwener/opencli`

预检可用性：

```bash
opencli xiaohongshu search "任意词" --limit 3 --format json
opencli xiaohongshu comments "https://www.xiaohongshu.com/search_result/test" --limit 2 --format json
```

结果判断：搜索成功+评论返回数据 → 环境就绪；搜索成功但评论空 → 提示用户登录 Chrome；完全失败（SECURITY_BLOCK）→ 切换 CDP 方案。

## 需求确认

与用户确认：监测对象、爬取目的、时间范围。

爬取目的决定采集策略：
- **广泛调研**：大量帖子 + 全量评论，产品/改版后摸清整体口碑
- **长时间线监控**：定期采集记录趋势，舆情监控、危机预警、竞品跟踪
- **事件影响评估**：事件前后对比，新功能上线、公关危机、营销活动

## 数据采集

**优先级**：深度测评帖 > 高赞评论(≥50赞) > 官方/宣传帖（单独标注，不计入用户舆情）

帖子类型：官方帖（品牌账号）单独统计；媒体/评测帖归类为"专业内容"；普通用户帖是情感分析主体。

**采集流程**：搜索关键词 → 获取帖子列表（含完整URL）→ 去重+按点赞排序 → 选择高价值帖子 → 依次获取详情+评论（每步间隔2-3秒）→ 汇总分析。

**请求频率**：每次操作间隔随机 1-3 秒，禁止并发，同一IP并发请求易触发 SECURITY_BLOCK。

**URL要求**：必须使用搜索结果中的完整URL（含xsec_token），禁止 bare note ID。

**数据校验**：SECURITY_BLOCK → 换用搜索结果中的完整URL重试；NOTE_NOT_FOUND → 跳过；EMPTY_RESULT → 可能需登录或帖子受限；连续3篇失败 → 降低频率或换方法。

## 评论分析

评论分析三原则：

**先读帖再读评。** 先获取标题、正文、标签了解背景，再看评论结合上下文判断。

**识别阴阳怪气。** 夸张语气可能是讽刺；"赚了-XXX"、"亏了XXX"是自嘲而非真负面；模仿炫富是在讽刺原帖；正话反说需结合整体语境判断。

**去噪。** 软广/营销号（频繁推产品带购买链接）单独标注不计入舆情；无具体观点的纯情绪杠精忽略。

## 报告模板

根据监测对象语义自动匹配：

- 含"vs"、"对比"、"竞品"、"选购" → `templates/调研报告.md`
- 单一品牌/产品、话题/事件 → `templates/舆情报告.md`

模板仅为参考，可根据调研到的有价值内容自行增减章节或创造新模板。

## 工具

**opencli**（首选）：内置适配器，自动处理反爬，输出JSON。

**CDP**（补充）：opencli完全失败时使用，处理登录墙或风控限制。
