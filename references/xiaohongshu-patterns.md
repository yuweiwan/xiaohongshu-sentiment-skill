## 平台特征

- **xsec_token 验证**：每个帖子 URL 都包含 `xsec_token` 参数，这是小红书的身份验证 token，丢失或过期会导致 `SECURITY_BLOCK`
- **搜索结果包含完整 URL**：搜索 API 返回的 URL 已包含有效 xsec_token，应直接使用而非提取 bare note ID
- **风控触发条件**：同一 IP 并发请求、请求频率过高、使用 bare note ID（无 token）都可能触发风控

## 有效模式

- **搜索 → 帖子详情/评论**：搜索结果中的 URL 包含完整 token，直接用于后续调用
- **延迟策略**：搜索 1-2 秒间隔，帖子详情/评论 2-3 秒间隔
- **单线程顺序请求**：避免并发，同一 IP 并发请求易触发 SECURITY_BLOCK
- **分批采集 + 去重**：热门关键词搜索结果可能有重复帖子，按 url 去重后分批采集
- **按点赞排序采集**：优先采集高赞帖子（≥50 赞），效率更高

## 已知陷阱

| 陷阱 | 表现 | 解决方案 |
|------|------|----------|
| 使用 bare note ID | `SECURITY_BLOCK: the note detail page was blocked by risk control` | 必须使用搜索结果返回的完整 URL |
| URL 无 xsec_token | `SECURITY_BLOCK` | 从搜索结果重新获取有效 URL |
| 并发请求 | `SECURITY_BLOCK` | 改为单线程顺序请求，延迟 2-3 秒 |
| 帖子已删除/受限 | `EMPTY_RESULT` | 跳过该帖子，换用其他帖子 |
| 请求频率过高 | 连续失败 | 增加延迟时间至 3-5 秒 |

## 错误码参考

| 错误码 | 含义 | 处理 |
|--------|------|------|
| `SECURITY_BLOCK` | 风控拦截，token 无效或请求异常 | 换用搜索结果中的完整 URL |
| `EMPTY_RESULT` | 页面加载无内容 | 可能需要登录或帖子已受限，跳过 |
| `NOTE_NOT_FOUND` | 帖子不存在或已删除 | 跳过该帖子 |

## 评论分析要点

- **必须先读帖子再读评论**：同一句话在赞美帖和批评帖中含义可能完全相反
- **小红书阴阳怪气比例高**：尤其在争议性话题下，高赞评论可能是讽刺而非认同
- **负数/亏损类评论**：如"赚了-100"，是自嘲讽刺，不是真正负面反馈
- **软广识别**：频繁推荐产品、带购买链接的账号需单独标注
